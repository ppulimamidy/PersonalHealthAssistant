# Insights Intelligence Gaps — Design Spec

> **Goal:** Close the remaining intelligence gaps in the Insights tab — make trends explain WHY, give each day a story, and connect predictions to treatment context.

---

## Gap 1: Trends Intelligence (P0)

### Current State
- 11 metric sparkline cards with latest value, trend arrow, average, source badges
- No explanation of why metrics changed
- MiniLineChart supports markers but trends.tsx doesn't pass any
- No cycle phase overlay, no medication markers, no experiment boundaries

### What We Build

**Trend Explanation API:**
`GET /api/v1/insights-intelligence/trend-explanations`

For each metric with sufficient data, generates a 1-sentence explanation:
```json
{
  "metrics": [
    {
      "metric": "hrv",
      "explanation": "HRV dipped 12% this week — you're in luteal phase, this is expected. Should recover in follicular.",
      "contributing_factors": ["cycle_phase", "sleep_quality"],
      "markers": [
        { "index": 5, "type": "experiment_start", "label": "Magnesium experiment" },
        { "index": 12, "type": "med_change", "label": "Started Metformin" }
      ]
    }
  ]
}
```

**Contributing factors from (cross-referenced):**
- Cycle phase (HRV, RHR, sleep, weight fluctuations)
- Medication starts/changes (glucose after Metformin, cholesterol after statin)
- Active experiments (metric tracking during experiment window)
- Nutrition patterns (sleep quality after high-sugar days)
- Symptom correlation (fatigue correlating with low HRV)
- Weather/season (future)

**Mobile changes:**
- Each MetricCard gets an explanation line below the chart
- MiniLineChart receives markers from the API (experiment, med, cycle)
- Tap to expand shows full explanation + all contributing factors

### API: `GET /api/v1/insights-intelligence/trend-explanations`

Gathers:
1. Timeline data (last 30 days of metric values)
2. Active medications with start dates
3. Active experiments with date ranges
4. Cycle phase info
5. Recent symptoms
6. Nutrition patterns

For each metric, determines the primary explanation:
- If cycle phase known and metric is HRV/RHR/sleep/weight → check if change aligns with expected phase shift
- If medication started in the period → check if metric trend changed after start
- If experiment active → note the experiment window
- Fallback: general trend interpretation

## Gap 2: Timeline Day Summaries (P1)

### Current State
- DayCards show wearable scores + action overlay (symptoms, meds, experiments)
- No synthesis — user must interpret raw numbers themselves

### What We Build

**Day Summary API:**
`GET /api/v1/insights-intelligence/day-summaries?days=N`

Returns per-day AI one-liner connecting wearable metrics + user actions:
```json
{
  "summaries": [
    {
      "date": "2026-03-20",
      "summary": "Solid sleep (8.1h) but low HRV — logged insomnia symptom. Your evening Metformin was taken."
    }
  ]
}
```

Generated by combining that day's:
- Sleep score/duration
- Steps/activity
- HRV/RHR
- Symptoms logged
- Medications taken/missed
- Meals logged
- Experiment status

**Mobile changes:**
- DayCard shows italic summary line at top (before scores)
- Expandable to full text if truncated

## Gap 3: Doctor Prep Enhancement (P1)

### Current State
- Generates a health report with score, key metrics, trends, concerns, improvements
- Doesn't include: medication list with lab evidence, supplement gaps, recommended tests, nutrition summary, cycle context

### What We Build

**Enhanced doctor prep context** — inject into the existing report generation:
- Active medications with lab evidence ("Metformin: A1C improved from 7.2 to 6.8")
- Supplement gaps from lab intelligence
- Recommended tests from lab intelligence
- Nutrition summary (avg daily intake, dietary adherence)
- Cycle context (regularity, current phase, phase-clustered symptoms)
